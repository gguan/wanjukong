# Wanjukong — 潮流玩具销售平台 PRD

## 1. 产品概述

### 我们在做什么
Wanjukong 是一个面向潮流玩具收藏爱好者的在线销售平台。用户可以按品牌、分类浏览和购买可动人偶、手办等潮流玩具。

### 目标用户

**买家 — 潮玩收藏爱好者**
- 关注特定品牌（Hot Toys、Threezero、DAM Toys 等）
- 关心商品比例（1/6、1/4、1/12）、版本差异（普通版/豪华版）
- 需要了解商品是现货还是预售，以及预计发货时间
- 希望快速下单，支持游客结账

**买家 — 注册用户**
- 希望保存收货地址，结账时自动填充
- 需要查看历史订单和订单状态
- 需要在下单后收到邮件确认

**卖家 — 店铺运营者（管理后台）**
- 需要管理商品信息、图片、变体和库存
- 需要管理品牌和分类体系
- 需要查看和处理订单，更新订单状态
- 需要了解销售数据和库存状况

### 产品定位
轻量级潮玩销售站，强调「快速浏览 → 立即购买」的简洁购买体验，支持 PayPal 在线支付和游客/注册双模式。

## 3. 页面设计

### 3.1 买家端页面

#### 首页 `/`
- **Hero 区域**：品牌标语 + 行动号召按钮（浏览商品 / 查看品牌）
- **品牌展示**：合作品牌 Logo 网格，点击进入品牌详情
- **精选商品**：展示 4 个精选商品卡片

#### 商品列表 `/products`
- **筛选栏**：品牌 / 分类 / 比例（1/6, 1/4, 1/12）/ 库存状态（现货/预售）
- **商品网格**：商品卡片展示封面图、品牌、名称、分类、比例、价格
  - 多变体商品显示「From $XX」起始价
  - 预售商品显示预售标识
- **筛选状态**：通过 URL 参数保持，支持分享筛选结果
- **空状态**：无匹配商品时显示提示 + 清除筛选按钮
- 🔴 **缺失**：无搜索功能、无分页/无限滚动

#### 商品详情 `/products/:slug`
- **图片画廊**：主图 + 缩略图列表，支持切换浏览
- **商品信息**：名称、品牌、分类、比例、库存状态
- **变体选择**：切换不同版本，显示各自的价格和封面图
  - 库存 ≤ 3 件时显示紧张提醒
  - 🔴 售罄变体仍可点击，无禁用/售罄标识
  - 🔴 变体的 specSummary 字段存在但未展示，用户无法比较不同版本包含的配件
- **商品描述**：纯文本展示
  - 🔴 不支持富文本/Markdown，多行描述会挤成一段
- **行动按钮**：「立即购买」→ 携带选中变体跳转结账页
- 🔴 **缺失**：无面包屑导航、无相关商品推荐、预售商品无预计发货时间展示

#### 品牌列表 `/brands`
- 所有品牌的网格展示，包含 Logo（或首字母头像）和品牌名

#### 品牌详情 `/brands/:slug`
- **品牌头部**：Logo、品牌名、商品数量
- **品牌商品**：该品牌下所有商品的网格展示
- 🔴 **缺失**：无筛选和排序（与商品列表页不同），商品多时浏览困难

#### 结账页 — 单品 `/checkout/:slug`
- **订单摘要**（右侧栏）：商品封面图、品牌名、商品名、变体名、数量、小计
- **变体选择**：多变体时显示单选切换
- **数量选择**：1-10 件
- **联系方式**：邮箱、姓名、电话（可选），支持登录提示和自动填充
- **配送地址**：国家下拉选择器、省/州、城市、邮编、详细地址
- **支付**：PayPal gate 模式 — 先验证表单 → "Continue to PayPal" → PayPal 按钮
- **库存校验**：PayPal 创建订单前校验库存
- **防重复提交**：PayPal SDK 内置 + 后端幂等检查

#### 结账页 — 购物车 `/checkout`
- 与单品结账相同的表单布局和 PayPal 流程
- **订单摘要**显示所有购物车商品
- 支持登录用户自动填充资料和默认地址

#### 购物车 `/cart`
- 商品列表、数量调整、删除
- 小计和总计
- 结账按钮

#### 订单确认 `/orders/:orderNo`
- **成功提示**：感谢信息 + 客户姓名
- **订单信息**：订单号、日期、订单状态、支付状态
- **商品清单**：商品图片、名称、变体、SKU、单价 × 数量 = 小计
- **金额合计**：小计 + 总计
- **收货信息**：收货地址 + 联系方式
- **导航链接**：「继续购物」+「查看订单历史」（仅登录用户）

#### 客户账户 `/account`
- **个人资料** `/account` — 查看/编辑姓名、手机，邮箱验证状态
- **地址管理** `/account/addresses` — 地址列表、新增/编辑/删除、设置默认
- **订单历史** `/account/orders` — 订单列表（订单号、日期、状态、金额），点击查看详情
- **订单详情** `/account/orders/:orderNo` — 完整订单信息，商品名可链接到商品页

#### 登录/注册/密码重置
- `/login` — 邮箱 + 密码登录
- `/register` — 邮箱 + 密码 + 姓名注册
- `/forgot-password` — 密码重置流程

## 4. 核心用户流程

### 4.1 购买流程（单品 Buy Now）
```
首页/商品列表
    ↓ 浏览、筛选
商品详情
    ↓ 查看图片、选择变体
    ↓ 点击「立即购买」
结账页
    ↓ 确认变体和数量
    ↓ 填写联系方式和收货地址
    ↓ 点击 "Continue to PayPal"
    ↓ PayPal 弹窗完成支付
    ↓ 后端捕获支付 → 创建订单（已付款）
订单确认页
    ↓ 查看订单详情
    ↓ 继续购物 / 查看订单历史
```

### 4.2 购买流程（购物车）
```
商品详情
    ↓ 加入购物车（可多件多变体）
购物车页
    ↓ 调整数量、删除
    ↓ 点击结账
结账页
    ↓ 填写联系方式和收货地址
    ↓ 库存校验 → PayPal 支付 → 创建订单
订单确认页
```

**设计决策：**
- **PayPal 在线支付**：所有结账流程均通过 PayPal 完成支付
- **游客 + 注册双模式**：游客通过 guestAccessToken 查看订单，注册用户在账户中查看历史
- **库存校验双重保障**：前端 PayPal 创建前校验 + 后端事务内原子扣减
- **幂等保护**：`captureAndCreateOrder` 检查 PaymentIntent 状态，防止重复创建订单

### 4.3 商品上架流程
```
创建商品（基本信息 + 封面图）
    ↓
上传多张商品图片 → 调整顺序 → 设置主图
    ↓
添加变体（普通版/豪华版等）→ 设置各自价格、库存、SKU
    ↓
将商品状态设为「上架」→ 前台可见
```

### 4.4 订单处理流程
```
买家完成 PayPal 支付 → 订单状态: 待处理 / 已付款
    ↓
卖家在后台查看订单列表，筛选待处理订单
    ↓
查看订单详情 → 确认订单（待处理 → 已确认）
    ↓
发货后更新状态 / 或取消订单
```

## 5. 商品体系设计

### 品牌
每个商品归属一个品牌（如 Hot Toys、Threezero）。品牌有独立展示页面，买家可按品牌浏览所有商品。

### 分类
商品按类型分类（如可动人偶、雕像、载具）。分类支持自定义排序。

### 比例
潮玩的核心属性，常见比例：1/6、1/4、1/12。作为商品属性存在，支持筛选。

### 变体
一个商品可以有多个变体版本，例如：
- **普通版** — 基础配件，较低价格
- **豪华版（Deluxe）** — 额外配件/地台，较高价格
- **限定版** — 限量发售

每个变体独立管理：价格、库存、SKU、封面图、上架状态、现货/预售状态。

**变体展示问题：**
- 🔴 变体的 `specSummary`（配置摘要）字段已有但前台未展示，用户看不到「豪华版多了什么」
- 🔴 预售变体的 `estimatedShipAt`（预计发货时间）未展示

### 商品状态
| 状态 | 含义 | 前台可见 |
|------|------|---------|
| 草稿 | 编辑中，未准备好 | 否 |
| 上架 | 正常售卖 | 是 |
| 下架 | 临时隐藏 | 否 |
| 售罄 | 库存为零 | 是（🔴 应显示售罄标识，当前未明确） |

### 库存类型
| 类型 | 含义 | 购买行为 |
|------|------|---------|
| 现货 | 有实物库存 | 下单即扣库存 |
| 预售 | 未到货 | 下单不扣库存，显示预计发货时间 |
