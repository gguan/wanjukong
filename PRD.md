# Wanjukong — 潮流玩具销售平台 PRD

## 1. 产品概述

### 我们在做什么
Wanjukong 是一个面向潮流玩具收藏爱好者的在线销售平台。用户可以按品牌、分类浏览和购买可动人偶、手办等潮流玩具。

### 目标用户

**买家 — 潮玩收藏爱好者**
- 关注特定品牌（Hot Toys、Threezero、DAM Toys 等）
- 关心商品比例（1/6、1/4、1/12）、版本差异（普通版/豪华版）
- 需要了解商品是现货还是预售，以及预计发货时间
- 希望快速下单，不想注册账号

**卖家 — 店铺运营者（管理后台）**
- 需要管理商品信息、图片、变体和库存
- 需要管理品牌和分类体系
- 需要查看和处理订单

### 产品定位
轻量级潮玩销售站，强调「快速浏览 → 立即购买」的简洁购买体验，不设购物车和用户账号体系。

## 3. 页面设计

### 3.1 买家端页面（共 7 页）

#### 首页 `/`
- **Hero 区域**：品牌标语 + 行动号召按钮（浏览商品 / 查看品牌）
- **品牌展示**：合作品牌 Logo 网格，点击进入品牌详情
- **精选商品**：展示 4 个精选商品卡片

#### 商品列表 `/products`
- **筛选栏**：品牌 / 分类 / 比例（1/6, 1/4, 1/12）/ 库存状态（现货/预售）
- **商品网格**：商品卡片展示封面图、品牌、名称、分类、比例、价格
  - 多变体商品显示「From $XX」起始价
  - 预售商品显示预售标识
- **筛选状态**：通过 URL 参数保持，支持分享筛选结果
- **空状态**：无匹配商品时显示提示 + 清除筛选按钮
- 🔴 **缺失**：无排序功能（按价格/上新时间）、无分页/无限滚动

#### 商品详情 `/products/:slug`
- **图片画廊**：主图 + 缩略图列表，支持切换浏览
- **商品信息**：名称、品牌、分类、比例、库存状态
- **变体选择**：切换不同版本，显示各自的价格和封面图
  - 库存 ≤ 3 件时显示紧张提醒
  - 🔴 售罄变体仍可点击，无禁用/售罄标识
  - 🔴 变体的 specSummary 字段存在但未展示，用户无法比较不同版本包含的配件
- **商品描述**：纯文本展示
  - 🔴 不支持富文本/Markdown，多行描述会挤成一段
- **行动按钮**：「立即购买」→ 携带选中变体跳转结账页
- 🔴 **缺失**：无面包屑导航、无相关商品推荐、预售商品无预计发货时间展示

#### 品牌列表 `/brands`
- 所有品牌的网格展示，包含 Logo（或首字母头像）和品牌名

#### 品牌详情 `/brands/:slug`
- **品牌头部**：Logo、品牌名、商品数量
- **品牌商品**：该品牌下所有商品的网格展示
- 🔴 **缺失**：无筛选和排序（与商品列表页不同），商品多时浏览困难

#### 结账页 `/checkout/:slug`
- **订单摘要**：商品封面图、名称、品牌、变体信息
  - 🔴 始终显示商品主图，未显示所选变体的封面图
- **变体选择**：单选按钮切换变体
  - 🔴 无库存状态显示，售罄变体仍可选
- **数量选择**：1-10 件
  - 🔴 上限固定为 10，不随实际库存调整
- **价格计算**：实时显示小计和总计
  - 🔴 无运费行，无税费行，小计=总计容易让用户误以为含运费
- **收货信息表单**：
  - 联系人：姓名、邮箱、电话（可选）
  - 收货地址：国家、省/州、城市、邮编、详细地址
  - 🔴 国家为文本输入（非下拉选择），输入不统一
  - 🔴 表单验证较弱，无手机号/邮编格式校验
- **提交订单**：验证后创建订单，跳转确认页
  - 🔴 未在提交前重新校验库存，可能超卖
  - 🔴 无防重复提交保护

#### 订单确认 `/orders/:orderNo`
- **成功提示**：感谢信息 + 客户姓名
- **订单信息**：订单号、日期、订单状态、支付状态
  - 🔴 所有订单显示「未支付」，无说明后续付款流程
- **商品清单**：商品图片、名称、变体、SKU、单价 × 数量 = 小计
- **金额合计**：小计 + 总计
- **收货信息**：收货地址 + 联系方式
- **继续购物**：返回商品列表的链接
- 🔴 **缺失**：无「接下来怎么做」引导、无邮件确认、离开页面后难以再次找到订单

### 3.2 管理后台页面（共 6 页）

#### 登录页
- 邮箱 + 密码登录表单
- 🔴 当前为 mock 认证，任意凭据可登录

#### 仪表盘
- 欢迎信息 + 系统健康状态检查
- 🔴 无销售数据、无订单概览、无库存预警

#### 商品管理
- **列表页**：商品表格（名称、品牌、分类、价格、状态），支持编辑和删除
  - 🔴 无搜索、无筛选、无排序、无分页
- **创建/编辑页**：
  - 基本信息：名称、Slug、品牌、分类、价格、库存、比例、描述
  - 状态设置：草稿 / 上架 / 下架 / 售罄
  - 库存类型：现货 / 预售
  - 封面图上传
  - 图片管理：批量上传、排序、设主图、删除
  - 变体管理：添加/编辑/删除变体，设置默认变体

#### 品牌管理
- 品牌表格 + 内联创建/编辑表单（名称、Slug、Logo URL）

#### 分类管理
- 分类表格 + 内联创建/编辑表单（名称、Slug、排序值）

#### 🔴 缺失页面
- **订单管理页**：后台侧边栏无订单入口，无法查看/处理订单
- **订单详情页**：无法更新订单状态（确认/取消/发货/标记已付款）

### 3.3 缺失的通用页面
- **关于我们**：介绍店铺背景和特色
- **联系方式**：客服/咨询渠道
- **运费与发货**：说明运费计算方式、发货时效
- **退换货政策**：退换规则
- **FAQ / 常见问题**
- **隐私政策 / 服务条款**（法律合规）

## 4. 核心用户流程

### 4.1 购买流程
```
首页/商品列表
    ↓ 浏览、筛选
商品详情
    ↓ 查看图片、选择变体
    ↓ 点击「立即购买」
结账页
    ↓ 确认变体和数量
    ↓ 填写联系方式和收货地址
    ↓ 提交订单
订单确认页
    ↓ 查看订单详情
    ↓ 继续购物 / 离开
```

**设计决策：**
- **无购物车**：潮玩单价高，用户通常单件购买，直接结账减少流程
- **无账号体系**：降低购买摩擦，用邮箱和订单号即可查询订单
- **无在线支付**：当前通过线下方式确认付款

**流程问题：**
- 🔴 从品牌详情进入商品详情后，返回链接固定指向 `/products` 而非来源的品牌页
- 🔴 订单确认页是买家唯一能看到订单的地方，离开后只能靠记住订单号手动拼 URL 访问
- 🔴 结账提交前不校验库存，在并发场景下可能超卖
- 🔴 无防重复提交，快速双击可能创建两笔订单

### 4.2 商品上架流程
```
创建商品（基本信息 + 封面图）
    ↓
上传多张商品图片 → 调整顺序 → 设置主图
    ↓
添加变体（普通版/豪华版等）→ 设置各自价格、库存、SKU
    ↓
将商品状态设为「上架」→ 前台可见
```

### 4.3 订单处理流程
```
买家提交订单 → 订单状态: 待处理 / 未支付
    ↓
🔴 卖家无法在后台查看和管理订单（缺失页面）
🔴 无订单状态流转功能（确认 → 已付款 → 已发货）
🔴 无通知机制（邮件/短信）
```

## 5. 商品体系设计

### 品牌
每个商品归属一个品牌（如 Hot Toys、Threezero）。品牌有独立展示页面，买家可按品牌浏览所有商品。

### 分类
商品按类型分类（如可动人偶、雕像、载具）。分类支持自定义排序。

### 比例
潮玩的核心属性，常见比例：1/6、1/4、1/12。作为商品属性存在，支持筛选。

### 变体
一个商品可以有多个变体版本，例如：
- **普通版** — 基础配件，较低价格
- **豪华版（Deluxe）** — 额外配件/地台，较高价格
- **限定版** — 限量发售

每个变体独立管理：价格、库存、SKU、封面图、上架状态、现货/预售状态。

**变体展示问题：**
- 🔴 变体的 `specSummary`（配置摘要）字段已有但前台未展示，用户看不到「豪华版多了什么」
- 🔴 预售变体的 `estimatedShipAt`（预计发货时间）未展示

### 商品状态
| 状态 | 含义 | 前台可见 |
|------|------|---------|
| 草稿 | 编辑中，未准备好 | 否 |
| 上架 | 正常售卖 | 是 |
| 下架 | 临时隐藏 | 否 |
| 售罄 | 库存为零 | 是（🔴 应显示售罄标识，当前未明确） |

### 库存类型
| 类型 | 含义 | 购买行为 |
|------|------|---------|
| 现货 | 有实物库存 | 下单即扣库存 |
| 预售 | 未到货 | 下单不扣库存，显示预计发货时间 |
